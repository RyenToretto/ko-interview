# 三皮肤 UI 全面美化

## 分析：当前 Hero 皮肤 vs 真实 LoL 客户端的差距

从截图分析英雄联盟客户端的核心视觉特征：

**已有但粗糙**：金色主色调、深蓝背景、菜单项高亮
**完全缺失**：

- 金色边框装饰线（顶栏、侧栏、卡片的金色描边/渐变边框）
- Logo 区域的盾形/菱形装饰元素
- 侧栏菜单的 icon 左侧金色竖条指示器
- 按钮的金色边框 + 悬浮发光效果（glow）
- 卡片和面板的斜角/切角装饰（clip-path）
- 顶部工具栏的分隔线金色渐变
- 整体的微光纹理（noise texture overlay）
- 滚动条自定义（深蓝+金色滑块）
- 标签/Badge 的金色边框风格
- 文字的金色光泽渐变效果（text gradient）

## 实施步骤

### Step 1: 整合两套设计规则到项目

将 Vercel Web Interface Guidelines 和 UI/UX Pro Max 的核心规则提取到 `.cursor/rules/` 中，作为后续开发的持久化参考。

- `.cursor/rules/web-design-guidelines.mdc` -- Vercel 规则精华（无障碍、表单、动画、排版、性能、暗色模式）
- `.cursor/rules/ui-ux-guidelines.mdc` -- UI/UX Pro Max 规则精华（配色、间距、触摸目标、状态反馈、导航）

### Step 2: 统一 CSS 变量体系

当前问题：许多组件硬编码颜色（如 QuestionCard 的 `#f8f9fa`、CodeRunner 的 `#1e1e2e`、QAQuestion 的 pre 背景）。

- [global.css](src/styles/global.css) 扩展变量集：新增 `--card-bg-secondary`、`--code-bg`、`--code-text`、`--code-border`、`--tag-`*、`--scrollbar-`* 等
- 修改 [QuestionCard.vue](src/components/QuestionCard.vue)、[QAQuestion.vue](src/components/QAQuestion.vue)、[CodeRunner.vue](src/components/CodeRunner.vue)、[CssRunner.vue](src/components/CssRunner.vue) 的硬编码颜色为 CSS 变量

### Step 3: Hero 皮肤 -- 英雄联盟客户端级别

重写 [skins.css](src/styles/skins.css) 的 Hero 部分，新增 [skins-hero.css](src/styles/skins-hero.css) 专属文件：

**侧边栏**：

- Logo 区域：金色边框底部 + 微光效果
- 菜单项激活态：左侧 3px 金色竖条 + 渐变背景 + 金色文字 + 微光
- 子菜单标题：金色分隔线
- 滚动条：深蓝轨道 + 金色滑块

**顶部工具栏**：

- 底部金色渐变分隔线（从两侧淡入中间亮）
- 按钮金色边框 + hover 发光
- 切换开关使用金色主色

**卡片和面板**：

- 金色 1px 边框（`border-color: #c89b3c33`）
- 卡片 header 底部金色渐变线
- 斜角装饰（CSS `clip-path` 在卡片右上角切一个小三角）
- hover 时边框加亮（transition 0.3s）

**按钮**：

- Primary 按钮：金色边框 + 深蓝背景 + hover 金色背景深蓝文字
- 文字按钮：金色文字 + hover 下划线

**标签/Tag**：

- 难度标签使用金色系色板而非 Element Plus 默认色
- 代码/问答标签使用 Hero 配色

**文字效果**：

- 标题文字金色渐变（`background-clip: text`）
- 描述文字使用 `#a09b8c`
- 链接/激活态使用亮金色 `#f0e6d2`

**全局装饰**：

- 自定义滚动条（所有可滚动区域）
- 微光纹理叠加层（CSS repeating-linear-gradient 制造细线纹理）
- 分隔线使用金色渐变（从透明到金色再到透明）

**首页特殊处理**：

- 标题使用金色渐变 + 大号 Georgia 字体
- 统计卡片金色边框 + hover 发光
- 时间线使用金色节点

### Step 4: Apple 皮肤 -- iOS 风格精致化

- 圆角统一 12px（卡片、按钮、输入框）
- 侧栏使用 SF 系统字体（`-apple-system, BlinkMacSystemFont`）
- 分组列表风格（圆角卡片内的紧凑列表）
- 搜索框圆角加大（20px 胶囊形）
- 毛玻璃效果加强（`backdrop-filter: saturate(180%) blur(20px)`）
- 微妙阴影（`box-shadow: 0 0 0 0.5px rgba(0,0,0,0.04), 0 2px 8px rgba(0,0,0,0.06)`）
- 暗色模式严格遵循 Apple Human Interface Guidelines 色彩

### Step 5: Default 皮肤 -- 现代简洁

- 基于 Element Plus 默认色但增加：
  - 统一间距节奏（8px 基准）
  - 卡片微妙阴影
  - hover 过渡动画（150ms ease-out）
  - 更好的代码块样式

### Step 6: 全局 UX 改进（基于两套 Guidelines）

从 Vercel 和 UI/UX Pro Max 规则中落实高优先级项：

- 所有 `<button>` / icon button 添加 `aria-label`
- 表单控件确保有 label
- 动画使用 `transform/opacity`，禁止 `transition: all`
- 侧栏菜单添加 `touch-action: manipulation`
- `prefers-reduced-motion` 媒体查询降级动画
- 滚动条美化（三套皮肤各自风格）
- 触摸目标最小 44x44px
