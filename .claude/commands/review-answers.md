# 审阅候选人答案

请审阅候选人在面试题中编写的代码实现。

## 审阅流程

1. **定位文件**：根据用户指定的题目名称或路径，找到对应的 `.vue` 文件
2. **读取代码**：读取候选人的完整实现
3. **对比要求**：根据题目描述中的要求逐一检查

## 评分维度

请从以下维度给出评分（每项 1-5 分）和具体评语：

### 代码题评分

| 维度 | 说明 |
|------|------|
| **正确性** | 是否满足题目要求，功能是否正常 |
| **代码质量** | 命名规范、代码结构、可读性 |
| **最佳实践** | 是否使用了推荐的写法和模式 |
| **完整性** | 是否覆盖了所有 TODO 任务 |
| **创新性** | 是否有额外的优化或巧妙实现 |

### 输出格式

```
## [题目名称] 审阅结果

### 评分
- 正确性: X/5
- 代码质量: X/5
- 最佳实践: X/5
- 完整性: X/5
- 创新性: X/5
- **总分: XX/25**

### 亮点
- ...

### 改进建议
- ...

### 综合评价
...
```

## 批量审阅

如果用户要求审阅所有题目，遍历 `src/views/` 下的所有代码题文件，检查 TODO 标记是否已被替换为实际实现。
